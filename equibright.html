Please use Left and Middle mouse buttons drag, as well as wheel, to get equal perceptual brightness for your display monitor. Just pressing these buttons makes some filtering. Values above zero (central line) will make impossible pure colors; while values below zero are reduces brightness (dynamic range) of picture, and may require OS support and/or higher brightness display monitor.

Equibright is impossible without one of both of these drawbacks. Amount of each in percent makes name of correction model; typical model for regular display will be like 90:10 to 80:20.

Then copy this into your byte-oriented HSL2RGB(h, s, l), before its math:

/* Equibright curve is hue->brightness func for fully saturate. */
int q[256] = {
}
/* Decompensate the curve for not full saturation. */
int p = q[h] * s * MIN(l, 255 - l) / (255*127);
l = l + p; /* Hope we never overflow uint8 here? */
